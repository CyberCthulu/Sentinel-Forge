# 🎬 Sentinel Forge — Demo Flow

## Objective

Demonstrate how fragmented cyber and physical signals are transformed into a **single, actionable decision**.

---

## Demo Sequence

### 1. System Idle

- Dashboard is calm
- No active threats
- Minimal event activity

---

### 2. Signal Emergence

Events begin appearing:

- Failed login attempts
- Additional failed attempts

**Narration:**
> “These are common signals—normally treated as noise.”

---

### 3. Escalation

- Successful login from an unfamiliar source
- Lateral movement across internal nodes

**Narration:**
> “Now we see escalation—but the signals are still fragmented.”

---

### 4. Cross-Domain Trigger

- Drone detected near perimeter (Sector B)

**Pause briefly to build tension.**

---

### 5. Fusion Moment (🔥)

System transitions:

**CRITICAL — Coordinated Intrusion Attempt**

- Confidence score appears
- Incident card becomes the dominant UI element

---

### 6. Explanation

System presents reasoning:

**Why this triggered:**

- Repeated authentication failures
- Successful login from an unfamiliar source
- Rapid lateral movement across systems
- Physical reconnaissance near perimeter

---

### 7. Action Layer

System provides immediate response actions:

- Lock affected accounts
- Isolate compromised node
- Dispatch patrol to Sector B
- Increase surveillance

---

## Key Narrative

> “We are not generating alerts.  
> We are producing a decision.”

---

## Demo Delivery Tips

- Do **not** rush transitions
- Pause before the incident appears
- Let the UI communicate the progression
- Speak calmly and deliberately
- Emphasize clarity over technical detail

---

## Closing Line

> “Sentinel Forge turns fragmented signals into clear defense.”
